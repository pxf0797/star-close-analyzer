# 拟合质量验证框架（T4 研究成果落地）

## 现状

当前系统仅在拟合时做两个二值检查：`|f'(0)|/price_scale < 0.05` 和 `f''(0) < 0`。没有残差分析、没有模型选择、没有样本外验证、没有回测级质量审计。以下框架填补这些空白。

---

## 1. 拟合质量门控系统（Fit Quality Gate System）

拟合后立即执行三级门控，任一阶段未通过则拒绝开仓：

### Stage 1：结构有效性（保留现有检查）

```python
if abs(deriv1_at_0) / max(price_scale, 1e-8) > 0.05:  # 拒绝
if deriv2_at_0 >= 0:                                   # 拒绝
```

### Stage 2：残差独立性（最高优先级新增）

对拟合残差 $\epsilon_t = y_t - \hat{y}_t$（t = 0, 1, ..., n-1）执行：

- **Durbin-Watson 统计量**：
  - $DW = \frac{\sum_{t=2}^{n} (\epsilon_t - \epsilon_{t-1})^2}{\sum_{t=1}^{n} \epsilon_t^2}$
  - 接受区间：**1.5 < DW < 2.5**（值 < 1.5 指示正自相关，值 > 2.5 指示负自相关）
  - 拒绝含义：残差存在序列依赖，轨迹预测不可靠

- **Ljung-Box Q 检验**（滞后阶数 h = min(5, n/4)）：
  - $Q = n(n+2) \sum_{k=1}^{h} \frac{\hat{\rho}_k^2}{n-k}$
  - 原假设 $H_0$：残差为白噪声
  - 接受条件：**p-value > 0.05**
  - 拒绝含义：残差中存在未被建模的自相关结构

> **预期效果**：过滤掉约 15-25% 的低质量拟合，这些拟合的三次多项式虽然导数合法，但轨迹根本不跟随实际价格走势。

### Stage 3：拟合质量（门槛指标）

- **CV(RMSE)**：$CV(RMSE) = \frac{RMSE}{\bar{y}} < 0.05$（与价格均值的归一化均方根误差）
- **R²**：$R^2 > 0.50$（拟合解释力须过半）
- **滚动窗口 CV 外推误差比**：留出最后 3-5 点的样本外 RMSE ÷ 样本内 RMSE **< 2.0**

---

## 2. 模型选择框架

当三次/二次多项式均可拟合时执行：

- **AICc（小样本校正 AIC）**：
  - $AICc = AIC + \frac{2k(k+1)}{n-k-1}$，其中 $AIC = n\ln(\frac{RSS}{n}) + 2k$
  - 参数计数：二次 k=3，三次 k=4
  - n=20 时校正项显著：三次比二次多 +0.47

- **决策规则**：
  - $\Delta AICc = AICc_{cubic} - AICc_{quadratic}$
  - 当 $\Delta AICc > 4$：拒绝三次项，选用二次（三次过拟合风险过高）
  - 当 $\Delta AICc \leq 4$：保留三次（额外的曲率带来了足够的信息增益）
  - BIC 作为保守二次校验：BIC 若同样偏好二次则更确信

> **预期效果**：在 n=20 的小样本窗口下，约 30-40% 的三次拟合实际不需要立方的复杂度。改用二次后样本外稳定性提升，尾端过度外推减少。

---

## 3. 综合质量评分（Composite Q-Score）

将多个二值/连续指标加权为单一分数 Q ∈ [0, 1]：

$$Q = 0.25 \cdot I(DW_{pass}) + 0.20 \cdot I(LB_{pass}) + 0.25 \cdot \max(0, 1 - \frac{CV_{RMSE}}{0.10}) + 0.20 \cdot \max(0, 1 - \frac{OOS/IS_{ratio}}{2.0}) + 0.10 \cdot \max(0, \frac{R^2 - 0.50}{0.50})$$

其中 $I(\cdot)$ 为指示函数（通过=1，否则=0）。

### 决策阈值

| Q 区间 | 判定 | 操作 |
|--------|------|------|
| Q < 0.50 | **拒绝** | 不开仓 |
| 0.50 ≤ Q < 0.75 | **边缘** | 只在信号强度极高（如 f'(0) 接近零、其他技术指标共振）时开仓 |
| Q ≥ 0.75 | **接受** | 正常开仓 |

### 实施要点

- Q-score 在拟合时一次性计算，写入 PositionState 或 Trade 记录
- 回测中可按 Q 分档分析表现（见第 4 节）
- 生产环境中可通过箱线图每月监控 Q 分布漂移

---

## 4. 回测级验证

### 分档分析

按 Q-score 将交易分为 [0, 0.5), [0.5, 0.75), [0.75, 1.0) 三档，每档统计：

- **胜率**（Win Rate）
- **夏普比率**（Sharpe Ratio）
- **盈亏比**（Profit Factor）
- **单笔平均盈亏**（Avg PnL per Trade）

预期趋势：Q 越高 → 胜率越高、夏普越高、回撤越低。若发现 U 型或倒挂分布，说明权重或阈值需要修正。

### 信息系数（Information Coefficient, IC）

- **定义**：每次交易的 Q-score 与标准化 PnL（PnL/ATR）之间的 Spearman 秩相关系数
- **阈值**：|IC| < 0.05 → Q-score 与收益无关联，框架无效
- **目标**：IC > 0.15（弱有效），IC > 0.25（强有效）

### 滚动重校准

- 每 **2-3 个月** 基于最近期数据重新校准门控阈值
- 校准方法：在当前窗口上最大化 IC + 交易频率的复合目标
- 避免过度优化：校准前后阈值变动超过 30% 应标记预警

## 5. 实施优先级

### Phase 1（立即 — 1-2 天）

- 在 `fit_cubic_trajectory()` 中添加 Stage 2（Durbin-Watson + Ljung-Box）
- 在 `fit_cubic_trajectory()` 中添加 Stage 3 的 CV(RMSE) 检查
- 失败时返回 None，保持调用方兼容

### Phase 2（短期 — 3-5 天）

- 实现 AICc 计算和模型选择逻辑（二次 vs 三次）
- 实现滚动窗口 CV 外推误差比
- 将选中的模型类型（cubic/quadratic）写入 PositionState

### Phase 3（中期 — 1-2 周）

- 实现 Composite Q-Score 和上述决策阈值
- 将 Q-score 写入 Trade 记录
- 实现回测分级统计 + IC 计算
- 添加预测区间：$\hat{y} \pm t_{\alpha/2, n-k} \cdot \hat{\sigma} \cdot \sqrt{1 + x^T(X^TX)^{-1}x}$

### Phase 4（持续 — 每月）

- Walk-forward 验证流程（每 2-3 月重校准）
- 引入 Diebold-Mariano 检验比较不同模型的预测精度
- 异常检测：监控 Q 分布均值/方差的月度漂移

## 预期综合效果

| 指标 | 当前基线 | 实施后预期 | 说明 |
|------|---------|-----------|------|
| 胜率 | ~45-50% | **50-60%** | +5-10pp |
| 夏普比率 | ~0.8-1.0 | **1.1-1.5** | +0.3-0.5 |
| 年化交易次数 | 100% | **60-80%** | 减少 20-40%（因低质量拟合被过滤） |

交易次数减少是**正面结果**而非代价：低质量拟合被拒意味着避免了对虚假模式的过度交易。每笔交易的期望收益和信噪比将提升。
